  "content": "# SessionTitle\n\n## Overview & Responsibilities\n\nThe `sessionTitle` module is a lightweight, standalone utility that generates AI-powered session titles using the Haiku model. It lives within **Infrastructure → CoreUtilities → SessionManagement** and serves as the **single source of truth** for AI-generated session titles across all surfaces of the application.\n\nThe module was intentionally designed with minimal dependencies to avoid pulling in heavy React/chalk/git dependency chains. This is critical because it is imported from the SDK print path (`print.ts`), where a lean import graph is required. Previously, session title generation was duplicated across `teleport.tsx` (for CCR remote sessions) and `rename/generateSessionName.ts` (for the `/rename` command); this module consolidates that logic for new callers.\n\n## Key Processes\n\n### Text Extraction Flow\n\nBefore generating a title, conversation messages must be flattened into a plain text string suitable as Haiku input:\n\n1. Iterate over the message array, keeping only `user` and `assistant` type messages\n2. Skip meta messages (`isMeta === true`) and non-human origin messages (`origin.kind !== 'human'`)\n3. Extract text from each message's content — handles both plain string content and content block arrays (filtering for `text`-type blocks)\n4. Join all extracted parts with newlines\n5. If the result exceeds 1000 characters, **tail-slice** to the last 1000 characters so recent context takes priority\n\n> Source: `src/utils/sessionTitle.ts:33-54`\n\n### Title Generation Flow\n\n1. Trim the input description; return `null` immediately if empty\n2. Call `queryHaiku()` with:\n   - A **system prompt** instructing Haiku to produce a 3–7 word sentence-case title\n   - A **JSON schema output format** constraining the response to `{ title: string }`\n   - The conversation text as the user prompt\n   - An `AbortSignal` for cancellation support\n   - Query metadata including `querySource: 'generate_session_title'` and the current session interactivity mode\n3. Extract text content from the Haiku response\n4. Parse the response as JSON via `safeParseJSON()`, then validate with a Zod schema (`z.object({ title: z.string() })`)\n5. Log a `tengu_session_title_generated` analytics event with the success status\n6. Return the trimmed title string, or `null` if parsing failed\n7. On any exception, log the error for debugging, fire a failure analytics event, and return `null`\n\n> Source: `src/utils/sessionTitle.ts:79-129`\n\n## Function Signatures\n\n### `extractConversationText(messages: Message[]): string`\n\nFlattens a message array into a single text string for title generation input.\n\n- **messages**: Array of conversation `Message` objects\n- **Returns**: A plain text string, at most 1000 characters (tail-sliced from the end)\n\n> Source: `src/utils/sessionTitle.ts:33-54`\n\n### `generateSessionTitle(description: string, signal: AbortSignal): Promise<string | null>`\n\nGenerates a sentence-case session title via the Haiku model.\n\n- **description**: The conversation text or session description to summarize\n- **signal**: `AbortSignal` for cancellation\n- **Returns**: A 3–7 word title string, or `null` on error/empty input/unparseable response\n\n> Source: `src/utils/sessionTitle.ts:79-129`\n\n## Configuration & Defaults\n\n| Constant | Value | Purpose |\n|---|---|---|\n| `MAX_CONVERSATION_TEXT` | `1000` | Maximum character length of conversation text sent to Haiku (tail-sliced) |\n\nThe system prompt enforces these title constraints:\n- **Length**: 3–7 words\n- **Casing**: Sentence case (capitalize only the first word and proper nouns)\n- **Format**: JSON response with a single `title` field\n\n## Edge Cases & Caveats\n\n- **Empty input**: Returns `null` immediately without making an API call if the trimmed description is empty\n- **Graceful failure**: All errors are caught and logged — the function never throws. Callers always receive either a valid title or `null`\n- **Tail-slicing**: For long conversations, only the last 1000 characters are sent. This means the title reflects recent context, not necessarily the original topic\n- **Dual-use path**: The module detects whether it's running in an interactive or non-interactive session via `getIsNonInteractiveSession()` and passes this through to the API call\n- **Legacy callers**: The older title generators in `teleport.tsx` and `rename/generateSessionName.ts` still exist for backwards compatibility; new code should use this module instead\n- **Zod validation via `lazySchema`**: The Zod schema is wrapped in `lazySchema()` to defer initialization, avoiding eager import-time evaluation costs\n- **Analytics**: Every title generation attempt (success or failure) fires a `tengu_session_title_generated` event"
