  "content": "# MicroModules\n\n## Overview & Responsibilities\n\nMicroModules is a collection of four small, focused files (approximately 65 lines total) spread across three utility subdirectories: `src/utils/github/`, `src/utils/memory/`, and `src/utils/todo/`. These files provide foundational type definitions and lightweight helpers that other parts of the system depend on. Within the architecture, they sit under **Infrastructure → CoreUtilities → DomainHelpers**, serving as the smallest building blocks consumed by the memory system, the todo/task tools, application state management, and telemetry bootstrapping.\n\nThe three concerns are:\n\n1. **Memory type definitions** — the canonical list of memory types and a git-repo detection helper\n2. **Todo item schemas** — Zod-based validation schemas for the todo list feature\n3. **GitHub CLI auth detection** — a subprocess-light helper that checks whether `gh` is installed and authenticated\n\n---\n\n## Memory Type Definitions\n\n### `src/utils/memory/types.ts`\n\nDefines the set of valid memory types as a const tuple and derives a TypeScript union from it.\n\n```ts\n// src/utils/memory/types.ts:3-12\nexport const MEMORY_TYPE_VALUES = [\n  'User',\n  'Project',\n  'Local',\n  'Managed',\n  'AutoMem',\n  ...(feature('TEAMMEM') ? (['TeamMem'] as const) : []),\n] as const\n\nexport type MemoryType = (typeof MEMORY_TYPE_VALUES)[number]\n```\n\n**Memory types**:\n\n| Type | Purpose |\n|---|---|\n| `User` | Per-user preferences and profile information |\n| `Project` | Project-level context and decisions |\n| `Local` | Machine-local state |\n| `Managed` | Remotely-managed memory entries |\n| `AutoMem` | Automatically extracted memories from conversations |\n| `TeamMem` | Team-shared memories (feature-gated behind `TEAMMEM`) |\n\nThe `TeamMem` value is conditionally included using the build-time `feature('TEAMMEM')` check from `bun:bundle`. When the feature flag is disabled at build time, `TeamMem` is dead-code eliminated entirely, so the runtime array and the `MemoryType` union both exclude it.\n\n**Consumer**: The compaction service (`src/services/compact/compact.ts`) imports `MEMORY_TYPE_VALUES` to enumerate memory types during context compaction.\n\n### `src/utils/memory/versions.ts`\n\nA synchronous helper that checks whether the current working directory is inside a git repository.\n\n```ts\n// src/utils/memory/versions.ts:6-8\nexport function projectIsInGitRepo(cwd: string): boolean {\n  return findGitRoot(cwd) !== null\n}\n```\n\nThis delegates to `findGitRoot()` which walks the filesystem (no subprocess spawned), making it safe for synchronous call paths. It is used by the `MemoryFileSelector` UI component to determine whether project-scoped memory options should be shown.\n\n---\n\n## Todo Item Schemas\n\n### `src/utils/todo/types.ts`\n\nDefines Zod validation schemas and inferred TypeScript types for the todo list feature.\n\n```ts\n// src/utils/todo/types.ts:4-18\nconst TodoStatusSchema = lazySchema(() =>\n  z.enum(['pending', 'in_progress', 'completed']),\n)\n\nexport const TodoItemSchema = lazySchema(() =>\n  z.object({\n    content: z.string().min(1, 'Content cannot be empty'),\n    status: TodoStatusSchema(),\n    activeForm: z.string().min(1, 'Active form cannot be empty'),\n  }),\n)\nexport type TodoItem = z.infer<ReturnType<typeof TodoItemSchema>>\n\nexport const TodoListSchema = lazySchema(() => z.array(TodoItemSchema()))\nexport type TodoList = z.infer<ReturnType<typeof TodoListSchema>>\n```\n\n**Key details**:\n\n- **`TodoItem`** has three fields:\n  - `content` — the task description (non-empty string)\n  - `status` — one of `'pending'`, `'in_progress'`, or `'completed'`\n  - `activeForm` — a non-empty string representing the current display form of the item\n- **`TodoList`** is simply an array of `TodoItem`\n- All schemas use `lazySchema()` (a project utility wrapping Zod in a lazy factory) to avoid circular initialization issues and defer schema construction until first use\n- Uses Zod v4 (`zod/v4`)\n\n**Consumers**:\n- `TodoWriteTool` (`src/tools/TodoWriteTool/TodoWriteTool.ts`) — uses `TodoListSchema` to validate input when Claude writes todo items\n- `AppStateStore` (`src/state/AppStateStore.ts`) — uses the `TodoList` type for application state\n- `RemoteAgentTask` (`src/tasks/RemoteAgentTask/RemoteAgentTask.tsx`) — uses the `TodoList` type to track remote agent task progress\n\n---\n\n## GitHub CLI Auth Status Detection\n\n### `src/utils/github/ghAuthStatus.ts`\n\nDetects whether the GitHub CLI (`gh`) is installed and authenticated, primarily for telemetry purposes.\n\n```ts\n// src/utils/github/ghAuthStatus.ts:4-7\nexport type GhAuthStatus =\n  | 'authenticated'\n  | 'not_authenticated'\n  | 'not_installed'\n```\n\n**Detection flow** (`getGhAuthStatus()`):\n\n1. **Check installation**: Calls `which('gh')` (uses `Bun.which` — no subprocess) to check if `gh` is on the PATH\n2. **If not found**: Returns `'not_installed'` immediately\n3. **Check authentication**: Runs `gh auth token` via `execa` with both stdout and stderr ignored\n4. **Return result**: Exit code 0 → `'authenticated'`, non-zero → `'not_authenticated'`\n\n**Design decisions**:\n- Uses `gh auth token` instead of `gh auth status` because the latter makes a network request to GitHub's API, while `auth token` only reads local config/keyring — faster and no network dependency\n- Pipes stdout to `'ignore'` so the actual auth token never enters the process's memory, avoiding accidental leakage through logs or crash reports\n- Sets a 5-second timeout and `reject: false` to gracefully handle slow or hanging `gh` processes\n\n**Consumers**:\n- `src/main.tsx` — called during application startup to record GitHub auth status in telemetry\n- `src/commands/remote-setup/remote-setup.tsx` — checked during remote setup to determine GitHub integration availability\n\n---\n\n## Edge Cases & Caveats\n\n- **`TeamMem` feature gate**: The `MEMORY_TYPE_VALUES` array contents differ between builds depending on the `TEAMMEM` feature flag. Code that iterates over this array will see different lengths in different builds — this is intentional for dead-code elimination but means the runtime type set is build-dependent.\n- **`projectIsInGitRepo` is synchronous**: It walks the filesystem to find `.git`, which is fast but blocking. The inline comment recommends `dirIsInGitRepo()` for async contexts.\n- **`gh auth token` timeout**: The 5-second timeout in `getGhAuthStatus()` means slow credential helpers (e.g., `gh` backed by a corporate SSO keyring) could cause a false `'not_authenticated'` result. Since this is used for telemetry rather than access control, the impact is limited to slightly inaccurate metrics."
