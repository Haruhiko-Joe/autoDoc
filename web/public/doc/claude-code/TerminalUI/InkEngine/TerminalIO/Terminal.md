It sits beneath the rendering pipeline and provides two core services:\n\n1. **Terminal type and write abstraction** — defines the `Terminal` type (stdout/stderr writable streams) and `writeDiffToTerminal`, which batches a list of `Diff` patches into a single atomic write, optionally wrapped in DEC 2026 synchronized-output markers to prevent visual flicker.\n\n2. **Capability detection** — a suite of functions that probe environment variables to determine what the host terminal supports: synchronized output, progress reporting, extended keyboard protocols, hyperlinks, scrollback clearing, and xterm.js identification. These are queried throughout the rendering engine to enable or disable features at runtime.\n\nThe module spans four files:\n- `src/ink/terminal.ts` — core type, diff writer, and most capability detectors\n- `src/ink/terminal-focus-state.ts` — focus/blur signal with `useSyncExternalStore`-compatible API\n- `src/ink/clearTerminal.ts` — cross-platform terminal clearing with scrollback support\n- `src/ink/supports-hyperlinks.ts` — OSC 8 hyperlink detection extending the `supports-hyperlinks` library\n\n## Key Processes\n\n### Writing a Diff to the Terminal\n\nThe `writeDiffToTerminal` function (`src/ink/terminal.ts:190-248`) is the single point through which the rendering pipeline flushes screen updates:\n\n1. If the diff list is empty, return immediately (no-op).\n2. Start a string buffer, prepending the BSU (Begin Synchronized Update) escape sequence unless `skipSyncMarkers` is true.\n3. Iterate over each patch in the `Diff` array, appending the appropriate content or escape sequence based on `patch.type`:\n   - `stdout` — raw content string\n   - `clear` — `eraseLines(count)` CSI sequence\n   - `clearTerminal` — delegates to `getClearTerminalSequence()`\n   - `cursorHide` / `cursorShow` — DEC private mode cursor sequences\n   - `cursorMove` / `cursorTo` — CSI cursor positioning\n   - `carriageReturn` — literal `\\r`\n   - `hyperlink` — OSC 8 link sequence\n   - `styleStr` — pre-encoded SGR/ANSI string\n4. Append ESU (End Synchronized Update) if sync markers are active.\n5. Write the entire buffer in a **single** `terminal.stdout.write()` call.\n\nBatching everything into one write minimizes syscalls and, combined with BSU/ESU, ensures the terminal renders the frame atomically.\n\n### Capability Detection Flow\n\nAll capability detectors follow the same pattern: check environment variables (and sometimes semver-parse `TERM_PROGRAM_VERSION`) against a known allowlist of terminals. Detection runs synchronously and results are typically cached at module load time (e.g., `SYNC_OUTPUT_SUPPORTED` is computed once at import).\n\nThe one exception is `isXtermJs()`, which combines a fast synchronous env check (`TERM_PROGRAM === 'vscode'`) with an async XTVERSION probe result. The probe is needed because `TERM_PROGRAM` is not forwarded over SSH, but the XTVERSION query/reply travels through the pty to the client terminal. `setXtversionName()` is called once from `App.tsx` when the probe response arrives on stdin.\n\n### Focus State Tracking\n\nFocus state (`src/ink/terminal-focus-state.ts`) uses a simple signal pattern:\n\n1. A module-level `focusState` variable holds the current state: `'focused'`, `'blurred'`, or `'unknown'`.\n2. `setTerminalFocused(v)` updates the state and synchronously notifies all subscribers.\n3. On blur, it also resolves any pending promises (via `resolvers`), enabling await-based blur detection.\n4. `subscribeTerminalFocus` / `getTerminalFocusState` form a `useSyncExternalStore`-compatible subscribe/getSnapshot pair for React integration.\n\n## Function Signatures\n\n### `writeDiffToTerminal(terminal: Terminal, diff: Diff, skipSyncMarkers?: boolean): void`\n\nBuffers all patches into a single write with optional BSU/ESU wrapping.\n\n- **terminal** — object with `stdout` and `stderr` `Writable` streams\n- **diff** — array of patch objects from the rendering pipeline\n- **skipSyncMarkers** — when `true`, omits DEC 2026 BSU/ESU wrapping (default `false`)\n\n> `src/ink/terminal.ts:190-248`\n\n### `isSynchronizedOutputSupported(): boolean`\n\nReturns `true` if the terminal supports DEC mode 2026. & Caveats\n\n- **tmux exclusion**: `isSynchronizedOutputSupported` returns `false` under tmux even though BSU/ESU pass through to the outer terminal. tmux breaks atomicity by chunking, so the sync markers provide no benefit and waste 16 bytes per frame.\n\n- **Windows Terminal and OSC 9;4**: Windows Terminal interprets OSC 9;4 as *notifications*, not progress indicators — `isProgressReportingAvailable` explicitly excludes it via `WT_SESSION`.\n\n- **SSH and TERM_PROGRAM**: `TERM_PROGRAM` is not forwarded over SSH, so env-based detection fails for remote sessions. `isXtermJs()` compensates with the XTVERSION probe, but the probe result arrives asynchronously. Early callers may miss it — the function should be called lazily (e.g., in event handlers) when SSH detection matters.\n\n- **`'unknown'` focus state**: Terminals without DECSET 1004 support never send focus events, so the state stays `'unknown'`. The module treats this as equivalent to `'focused'` (`getTerminalFocused()` returns `true`), avoiding unnecessary throttling.\n\n- **Legacy Windows console**: `getClearTerminalSequence` falls back to screen-only clear (no scrollback) with the HVP escape (`CSI 0 f`) for legacy consoles that don't support `ESC[3J`.\n\n- **Extended keys caution**: The `EXTENDED_KEYS_TERMINALS` allowlist exists because unconditionally enabling Kitty keyboard protocol caused input parsing failures in xterm.js-based terminals and over SSH (#23350).\n\n- **`writeDiffToTerminal` is all-or-nothing**: The entire diff is buffered into a single string and written in one call. If the diff is huge, this means a single large allocation, but it guarantees atomic output."
